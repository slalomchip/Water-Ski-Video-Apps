Introduction

 

Video Tape Measure (VTM) is a water ski, sit ski, and barefoot jump measuring software application that uses the same principles and math model as Video Jump and SplashEye. That means that the concepts of geometric relationships between the ramp, grid buoys, and cameras remain the same, including the 30o field of view constraint unless you are using low distortion lenses or the camera corrects for lens distortion.  VTM runs on a 64-bit Windows 10 platform and accepts either standard definition or high definition video streams. All VTM operations are on a single screen.

 

VTM is integrated in with WaterSkiConnect for seamless transfer of skier information and jump scores between VTM and the scoring system.

 

There are three advantages to using high definition cameras for videoing the jump; (1) Given the same camera location and zoom as a standard definition camera, the high definition signal provides a crisper image (2) grid box sizes can be widened if the 30-degree field of view constraint is still satisfied (move the cameras farther from the landing zone), and (3) the grid buoys do not need to fill out the screen as much as when using a standard definition camera because of more pixels throughout the image. In fact, the checks for filing out the screen are not in VTM. Rather, VTM presents the tournament tolerance levels upon successful grid setup and the user determines whether the setup meets tournament requirements.  Whereas VTM can accept a high definition video feed, it will not provide full HD 1920x1080 resolution on the screen because of available screen space for both the video stream and the controls.  VTM scales a 16:9 aspect ratio video stream (such as 1920x1080) to 1337x752 pixels.  Likewise, VTM will scale 4:3 aspect ratio video stream (such as a 640x480) video stream to 1002x752 pixels.

 

Speaking of screen resolution, VTM was developed on a laptop computer with a screen resolution of 1920x1080 resolution that was scaled to 125%.  These display settings fill out the screen nicely. You might want to adjust your display settings to match these settings.  Unlike Video Stopwatch, the VTM video window does not scale to fit the overall screen size.

 

VTM accommodates up to four grid setups but can display only one grid at a time.  To have multiple grids displayed concurrently, multiple computers are required.

 

VTM includes a personal video recorder (PVR). The PVR is a quick and convenient way to replay a jump without the need to replay from a DVR or VHS player in the event a jump was missed.  However, be aware that, if you are using a multi-grid setup, PVR is only recording the active grid. Therefore, a DVR will be necessary to replay a jump-out-of-grid scenario.

 

The computer must have the users Documents folder available. VTM will not run without the folder present.

Framing the Grid in the Video Window

 

It is suggested that the grid be vertically centered within the video. Because the VTM video window is void of overlaid text boxes (except at the topmost of the video window), there is no need to tilt the camera so the near grid buoys are at the bottom of the mage. Vertically centering the grid helps minimize any lens distortion effects at the bottom edge of the video image.

 

 

Operations

 

Configuration Settings

The configurations settings window is presented when VTM is first launched. Verify or modify any settings. This window may be revisited at any time during operations and the changes will take effect immediately without needing to restart VTM.

 

VTMs default directory for saved data is in C:\Users\(youcomputerusername)\Documents.  If you have Microsoft OneDrive installed, the Documents folder in the left pane of Windows Explorer may point to C:\Users\(youcomputerusername)\OneDrive\Documents.  These are two different directories.  If you need to access files in the VTM directory, make sure you navigate to the correct Documents directory.

 

Grid Setup

The grid setup frame on the right side of the screen is either red or green. Red means that the grid is not setup and green means that it is. The grid must be setup prior to measuring jumps.

 

The first grid setup is performed using a video stream. Thereafter, a grid may be set up from a video stream or a frozen frame.  Grid setup can be entered at any time.

 

A grid loaded from a file is assumed to meet the setup checks. However, it may need adjustment because of small changes due to twisting camera poles, etc.  If loading a grid from a file, always visually check the marker placements and adjust if necessary.

 

When a grid setup passes the quality check, a 11x11 grid of homologation levels is presented. Make sure that the grid meets the tournament sanction level across the entire landing zone. The homologation levels are R (satisfies class R tournament requirements, L (satisfies class L and E tournament requirements), C (satisfies class C tournament requirements), and X (does not satisfy class C tournament requirements). Uncheck the Show Homologation Level checkbox to clear the grid.

 

A matrix of one-pixel adjustments is displayed when checking the Show Grid Accuracy checkbox. A matrix of distances across the grid is displayed when the Show Distances checkbox is checked.

 

Using radio buttons in the setup frame, you can toggle the displayed coordinates between the survey coordinates and the screen coordinates.  This feature is enabled only after the grid is set up.

 

Save the grid after any adjustments are made.  Switching grids reverts to the last saved grid, not the last grid that passes the setup check.  Always saving the last adjustment that passes the setup check is good practice so that the last setup is preserved when VTM is restarted.

 

Measuring Jumps

The large button at the top of the window is multifunctional. Use this button when the skier hits the ramp. Use this button again record the jump distance.  This button takes keyboard focus. Pressing the TAB key once or twice will bring focus to the button. When the button has focus, the ENTER key, RETURN key, or spacebar can be used in lieu of clicking the button. This is especially convenient when the skier hits the ramp because the users eyes are focused on the skier rather than the screen.  These keys can also be used to record the jump distance.

 

PVR

The PVR enables the user to quickly jump 60 seconds back and then select a frame to measure on if a jump was missed.  If the PVR is enabled, then the PVR records at the same frame rate as when capturing jumps (which is set in the configurations window).

 

If the PVR is checked in the Configurations window, then an orange PVR button appears on the lower left of the main window and the video stream is being recorded. If the orange PVR button is not present, the video is not being recorded. The stored video can consume several gigabytes of storage during a tournament, so make sure that you review available storage at the completion of each tournament and delete or archive old videos as needed.

 

When you press the PVR button, the last two minutes of video is grabbed and presented in the video window with a thick orange border around it. The frame displayed is the middle of the video (60 seconds from the beginning and 60 seconds from the end). Use the slider bar to quickly find the jump, then frame-by-frame to select the frame you want to use for measuring the jump.

 

Clicking the Next Jump button resumes the live video stream.

 

The recorded video is stored in a file with a datetime stamp as the file name (yyyymmdd_hh-mm-ss.mp4).  Each time the user presses the PVR button, the current file is closed and a new file created with a new datetime stamp before the recorded video is displayed within the orange border.

 

You should also record the video to a DVR or video tape regardless of whether you are using the PVR.  A backup capture device serves as a safety net if there is a computer crash, operator error, the VTM was receiving the wrong video stream, or some other unforeseen situation.

 

Verify

The verify tool is enabled after the grid is set up. Use verify for more grid setup checks. Verify provides X, Y coordinates of a measured point. Use these coordinates to compare with X, Y coordinates from a course survey. Also, the tournament technical controller should use this tool for providing the X, Y coordinates in the homologation dossier. When the Verify tool is enabled, the X, Y coordinates in the Verify frame are displayed whenever the mouse is clicked within the video frame. Verify is disabled during grid setup and after a skier hits the ramp. Verify is enabled after the grid is set up and after the user selects Next Jump.

 

WaterSkiConnect

When both VTM and the scoring system are connected to WaterSkiConnect, the scoring system will automatically send the jumpers name, round number, and pass number for every pass.  VTM sends the measured jump distance to the scoring sysem at the same time the distances are displayed. 

 

The scorer can provide the information needed to connect VTM to WaterSkiConnect.  The Event ID (tournament sanction mumber) is mandatory.  If the tournament is at a multi-lake site and each lake has its own scoring system, then the Event Sub ID is also needed to distinguish the different scoring systems from one another.  The connection information entered into VTM must match the scoring systems information, otherwise the data will not transfer between the two systems. 

 

WaterSkiConnect furnishes a PIN once connected.  Make sure that the PINs issued to VTM and the scoring system match.  If they are different, the two systems are not connected to each other and you will need to disconnect from WaterSkiConnect and then reconnect.

 

VTM displays the jumpers name, pass, and round in white font on a black background when the information s received via WaterSkiConnect and this information cannot be changed.  If you need to alter the skier name, pass, or round, you must first disconnect from WaterSkiConnect.

 

Skier Names

A list of skier names can be imported from WSTMS. Ask the scorer to bring up the jump running order in WSTMS and then click on Video Run Order. A file will be exported. Load that file using the menu system at the top of the VTM screen. VTM extracts the last name, first name, and skier division and adds them to a list. Names that are already in the list are not duplicated. It is suggested that the list be deleted at the beginning of each season because some skiers age up to a new division.

 

Names can also be added to the list by importing a comma separated values (.csv) file or a text (.txt) file.

·       First line in the file is the header row.  VTM does no interpre the first row as a skiers name.  Suggested values in the first lines are Skier Name, Name, or similar.

·       Each subsequent line in the file should be one name.  Suggested name format is Lastname, Firstname.  VTM will not try to interpret the first and last names  VTM takes the names exactly as typed, alphabetizes them, and adds them to the dropdown list.

 

Importing skier names will not duplicate the name in the list  if VTM finds another exact match, the name is not added. 

 

You can also type a name into the names field in the main window. To add a typed name to the list, make sure to press the ENTER key after typing the name.

 

Loading Pictures and Videos from Files

Previous jumps that were saved to videos and/or pictures can be loaded via the menu system at the top of the main window. A valid setup must exist prior to loading these files, regardless of whether the current grid is the same as the grid used in the videos/pictures. A new grid setup can occur after a file is loaded.

 

The Loupe Window

The loupe window magnifies a small portion of the screen. View the Loupe when adjusting the markers. The loupe magnification can be adjusted using the radio buttons to its left.

 

Video Enhancements

The brightness and contrast on the live video can be adjusted with the sliders in the bottom right corner of the main window. Brightness and contrast adjustments cannot be made to videos or pictures that were already recorded. The adjustments to the live video are applied to the recorded video captures and saved pictures.

 

External Distance Display

VTM features a distance display capability for displaying jump distances to spectators, jumpers, scorer, or announcer.  There are four options for the distance display.  Connect a second monitor to your computer and configure it as an extended monitor.  Then enable the distance display, drag it into the screen space of the extended window and then maximize the distance display.  The display will update whenever the meters and feet text boxes in the main window update.  When the skier hits the ramp for the next jump, the display clears. The four options for display are:

·       Flash Mob  alternates the display between meters and feet every 1.5 seconds

·       Two Peas in a Pod  Displays both meters and feet distances at the same time, as well as the skiers name

·       Meter-O-Rama  Displays meters

·        Bigfoot  Displays feet

 

Keyboard Shortcuts

Several keyboard shortcuts are available to streamline operations:

·       Right and left arrow keys  These will either move the markers right and left or move captured video frames right forward and backward, depending on where you are in the workflow

·       Up and down arrow keys  these will move the markers up or down

·       Tab key Will either bring or remove focus to the Skier Hit Ramp or Record button, or it will sequence you through entering survey coordinated, depending upon where you are in the workflow

·       Enter and Return keys  Either key will substitute for a click of the Skier Hit Ramp or Record button (depending on which is present) if the button has focus

·       R key  A shortcut key to record the jump distance

·       C key  A shortcut key to capture a frame

·       N key  A shortcut key to go to the next jump

·       S Key  A shortcut key to reselect the frame prior to recording the jump distance
